# Global Development Standards

## Communication Standards
- **Daily Sync**: Brief standup focusing on blockers and dependencies
- **Documentation**: Maintain up-to-date README, API docs, and component documentation
- **Code Reviews**: All agents participate in reviewing code within their expertise
- **Knowledge Sharing**: Share discoveries, patterns, and solutions across the team

## Quality Assurance
- **Definition of Done**: Feature is tested, documented, and meets all acceptance criteria
- **Performance Standards**: All features meet Core Web Vitals benchmarks
- **Security Review**: Each feature undergoes security review before deployment
- **Accessibility Check**: All UI components pass accessibility audit

## Emergency Protocols
- **Hotfix Process**: Clear protocol for critical production issues
- **Rollback Procedures**: Documented steps for rolling back problematic deployments
- **Incident Response**: Defined roles and responsibilities during production incidents
- **Communication Chain**: Clear escalation path for critical issues
